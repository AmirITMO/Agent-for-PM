# Agent 3 — PM-трекер задач

Мультиагентная система управления задачами для команды MarketAI.
Telegram-бот + веб-дашборд + автоматические уведомления.

---

## Архитектура

```
Telegram-бот (python-telegram-bot)
       │
       ├── NL-парсер запросов (query_parser.py)
       ├── Форматирование ответов (formatter.py)
       └── PostgreSQL ←──── FastAPI веб-дашборд (web.py)
                │
                └── APScheduler (scheduler.py)
                     ├── Утренняя сводка (09:00)
                     └── Проверка дедлайнов (каждые 30 мин)
```


## Структура базы данных

| Таблица | Назначение |
|---------|-----------|
| `users` | Сотрудники (имя, telegram_id, роль admin/employee) |
| `projects` | Проекты (MarketAI Dev, MarketAI Marketing) |
| `tasks` | Задачи (название, статус, приоритет, исполнитель, оценка, дедлайн) |
| `task_comments` | Комментарии к задачам |
| `notification_log` | Лог отправленных уведомлений (дедупликация) |

### Статусы задач (канбан-колонки)

Backlog → Planning → To Do → WIP → Done → Approved → Hold

### Приоритеты

- **Обычный** — стандартная задача
- **Срочный** — требует немедленного внимания
- **Баг** — ошибка, которую нужно исправить

## Запуск

### Требования
- Python 3.12+
- Docker (для PostgreSQL)
- Telegram Bot Token (от @BotFather)

### Установка

```bash
pip install -r requirements.txt
```

### Локальный запуск

```bash
# 1. Запустить PostgreSQL
docker run -d --name agent3-postgres \
  -e POSTGRES_DB=agent3_pm \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 postgres:16-alpine

# 2. Заполнить тестовыми данными
python -m agent3_pm.seed

# 3. Запустить всё (бот + веб + шедулер)
export TELEGRAM_BOT_TOKEN=your_token_here
python -m agent3_pm.main
```

### Docker Compose (продакшен)

```bash
TELEGRAM_BOT_TOKEN=your_token docker-compose up -d
```

### Переменные окружения

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `TELEGRAM_BOT_TOKEN` | — | Токен бота (обязательно) |
| `DATABASE_URL` | `postgresql+asyncpg://postgres:postgres@localhost:5432/agent3_pm` | Подключение к БД |
| `WEB_BASE_URL` | `http://localhost:8080` | Базовый URL веб-дашборда |
| `TIMEZONE` | `Europe/Moscow` | Часовой пояс |
| `MORNING_SUMMARY_HOUR` | `9` | Час отправки утренней сводки |
| `DEADLINE_CHECK_INTERVAL_MINUTES` | `30` | Интервал проверки дедлайнов (мин) |
| `DEADLINE_WARNING_HOURS` | `24` | За сколько часов предупреждать о дедлайне |

## Веб-дашборд

Доступен по адресу `http://localhost:8080`.

### Страницы

| URL | Описание |
|-----|----------|
| `/admin` | Канбан-доска — все задачи по колонкам, фильтр по проекту |
| `/admin/tasks` | Табличный список всех задач с фильтрами |
| `/admin/users` | Управление сотрудниками (добавить, редактировать, удалить) |
| `/my/{user_id}` | Личная доска сотрудника — только его задачи |
| `/task/{id}` | Детальная карточка задачи |

### Функции веб-дашборда

- Канбан-доска с колонками по статусам
- Фильтрация по проектам
- Создание задач через модальное окно (проект, исполнитель, статус, приоритет, часы — обязательные поля; дедлайн — необязательный)
- Смена статуса задачи через dropdown прямо на карточке
- Визуальные метки: красная рамка для просроченных, желтая для горящих
- Бейджи: «Просрочено», «Горит», «Срочный», «Баг»
- Управление сотрудниками: добавление, редактирование (имя, роль, Telegram ID), удаление

## Telegram-бот

### Регистрация сотрудника

Два способа попасть в систему:

1. **Админ создаёт** сотрудника через веб-панель `/admin/users`, затем сотрудник пишет боту:
   ```
   /bind Амир
   ```
   Бот находит непривязанный аккаунт «Амир Хайруллин» и привязывает Telegram.

2. **Самостоятельная регистрация:**
   ```
   /register Амир Хайруллин
   ```
   Бот создаёт аккаунт и сразу привязывает Telegram.

### Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и инструкция |
| `/help` | Список всех команд |
| `/register Имя Фамилия` | Зарегистрироваться |
| `/bind Имя` | Привязать Telegram к существующему аккаунту |
| `/today` | Мои задачи на сегодня |
| `/overdue` | Просроченные задачи по всей команде |
| `/status Dev` | Статус проекта с прогресс-баром |
| `/task 5` | Детали задачи #5 |
| `/summary` | Сводка по команде (только для админов) |

### Свободный текст (NL-запросы)

Бот понимает запросы на естественном языке:

- «что у меня на сегодня» → список задач пользователя
- «мои задачи» → то же самое
- «что просрочено» → просроченные задачи всей команды
- «статус проекта Dev» → прогресс проекта
- «#5» или «задача 5» → детали задачи
- любой другой текст → поиск задачи по названию

### Проактивные уведомления

Система автоматически:

- **Каждое утро в 9:00** отправляет руководителю (admin) сводку: просрочки, горящие сегодня, открытые задачи по сотрудникам
- **Каждые 30 минут** проверяет дедлайны и отправляет исполнителям предупреждения о горящих/просроченных задачах
- Уведомления **не дублируются** — каждый тип уведомления отправляется максимум 1 раз в день на задачу

## Логика работы с задачами

### Что считается «просроченным»

Задача просрочена, если:
- У неё есть дедлайн
- Дедлайн прошёл (дата < сегодня)
- Статус активный (Backlog, Planning, To Do, WIP)
- Задачи со статусом Done, Approved, Hold **не** считаются просроченными

### Что считается «горящим»

Задача горит, если:
- Дедлайн сегодня или завтра
- Статус активный
- Ещё не просрочена

### Задачи на сегодня (для сотрудника)

Показываются задачи, которые:
- Назначены на этого сотрудника
- Имеют активный статус
- Либо просрочены, либо дедлайн сегодня, либо в статусе TODO/WIP

Просроченная задача показывается **только** в секции «Просрочено», не дублируется в «В работе».

**Цель:** Проверить полный цикл регистрации.

4. Бот подтверждает регистрацию

6. Проверить что на `/admin/users` появился новый сотрудник «Иван Петров» со статусом «Привязан»

### Тест 2 — Привязка к существующему аккаунту
