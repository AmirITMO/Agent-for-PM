# Agent 3 — PM-трекер для MarketAI

## Что это

Система управления задачами для команды. Два интерфейса:
- **Веб-дашборд** — канбан-доска для руководителя
- **Telegram-бот** — для сотрудников, задачи в одно нажатие

Руководитель видит все задачи, назначает исполнителей, контролирует дедлайны.
Сотрудник видит только свои задачи и отмечает выполнение.

---

## Демо-сценарий

### 1. Вход руководителя

Открываем веб-платформу. Страница входа — имя и пароль.
Руководитель попадает на канбан-доску.

Что видит:
- Отдельные доски по проектам (MarketAI Dev, MarketAI Marketing)
- Колонки: Бэклог → Планирование → К выполнению → В работе → Готово → Принято → На паузе
- Карточки задач с приоритетами (баг, срочный, обычный)

### 2. Работа с задачами

Клик на карточку — открывается редактирование:
- Название, описание, приоритет, оценка в часах, дедлайн
- Назначение исполнителя
- Кнопка «Выполнено» — закрывает задачу и шлёт уведомление руководителю в Telegram

Кнопка «+ Добавить задачу» внизу каждой колонки — задача сразу попадает в нужный статус и проект.

### 3. Добавление сотрудника

Раздел «Сотрудники» → «+ Добавить сотрудника»:
- Указываем имя и должность (МОП, Маркетолог, Программист, Авитолог и т.д.)
- Система генерирует invite-код
- Руководитель отправляет код сотруднику

21 должность: CEO, COO, CMO, CTO, Коммерческий директор, РОП, МОП, Программист, Продуктолог, Маркетолог, Директолог, ВК, Яндекс, ТГ, Авитолог, СММ, Амбассадор, Партнер, Диагност, Отдел заботы, Тех. поддержка.

### 4. Подключение сотрудника через Telegram

Сотрудник открывает бота → /start → вводит invite-код.
Бот привязывает аккаунт. Появляются кнопки:

- **Мои задачи** — список назначенных задач
- **Просрочки** — что просрочено по команде
- **Статус проектов** — выбрать проект, увидеть прогресс

Бот понимает и свободный текст: «что у меня на сегодня», «что просрочено», «статус проекта Dev».

### 5. Автоматические уведомления

Система работает без участия руководителя:
- **Каждое утро в 9:00** — сводка: просрочки, горящие сегодня, открытые задачи по сотрудникам
- **Каждые 30 минут** — проверка дедлайнов, предупреждение исполнителю
- **При нажатии «Выполнено»** — уведомление руководителю с карточкой задачи

Не спамит — одно уведомление по задаче в день.

### 6. Настройки

Раздел «Настройки» на веб-платформе или команда /settings в боте:
- Час утренней сводки
- Интервал проверки дедлайнов
- За сколько часов предупреждать
- Часовой пояс

## Разделение доступа

| | Руководитель | Сотрудник |
|---|---|---|
| Видит все задачи | да | нет |
| Видит свои задачи | да | да |
| Создает задачи | да | нет |
| Назначает исполнителей | да | нет |
| Меняет статус своих задач | да | да |
| Нажимает «Выполнено» | да | да |
| Управляет сотрудниками | да | нет |
| Настройки уведомлений | да | нет |
| Получает утреннюю сводку | да | нет |

## Стек

- Python, FastAPI, PostgreSQL
- Telegram Bot API (python-telegram-bot)
- APScheduler для автоматических уведомлений
- Docker для развертывания
