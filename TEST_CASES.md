## Регистрация и вход (1-3)

**TC-1. Регистрация нового пользователя**
1. /start → «Зарегистрироваться» → ввести имя → выбрать должность
2. Должности сгруппированы: 1 уровень / 2 уровень / 3 уровень
- Ожидание: меню с 5 кнопками (Мои задачи, Просрочки, Задать задачу, Спросить по задачам, Инструкции)

**TC-2. Вход в веб по ссылке из бота**
1. Нажми «Мои задачи» → скопируй ссылку «Открыть трекер»
2. Открой в браузере
- Ожидание: автоматический вход, канбан-доска

**TC-3. Кнопка «Инструкции»**
1. Нажми «Инструкции»
- Ожидание: описание всех функций включая групповые чаты, автоматику из БЗ, напоминания

## Создание задач через бота (4-9)

**TC-4. Текстовая задача — агент уточняет доску и этап**
1. «Задать задачу» → «Сделать лендинг для Амира P0»
2. Агент спрашивает: «На какую доску?» (с кнопками)
3. Выбери доску → агент спрашивает этап (с кнопками)
4. Выбери этап → карточка + файлы? → «Нет, создать»
- Ожидание: задача на канбане, ссылка кликабельная «Открыть задачу»

**TC-5. Голосовая задача**
1. «Задать задачу» → отправь голосовое
2. Бот: «Распознаю...» → «Распознано: ...» → уточняет доску/этап
- Ожидание: задача создана из голоса

**TC-6. Задача с файлами**
1. Создай задачу → на «Прикрепить файлы?» нажми «Да»
2. Отправь фото → «Готово»
- Ожидание: фото в комментариях на странице задачи

**TC-7. Агент НЕ создаёт без доски и этапа**
1. «Задать задачу» → «Проверить отчёт»
- Ожидание: агент спрашивает доску, НЕ создаёт автоматически

**TC-8. «Задай задачу амиру» — это задача, не напоминание**
1. «Задать задачу» → «задай задачу амиру написать мише»
- Ожидание: create_task (не set_reminder), уточняет доску

**TC-9. Уведомление исполнителю при назначении**
1. Создай задачу и назначь на зарегистрированного пользователя
- Ожидание: исполнитель получает в личку «Тебе назначена задача от ... — Открыть задачу»

## Автоматическая подгрузка задач из базы знаний (29-35)

**TC-29. KB Watcher инициализируется без спама**
1. Перезапусти контейнер: `docker-compose restart agent3-app`
2. Проверь логи: `docker-compose logs --tail=10 agent3-app`
- Ожидание: «KB watcher initialized user/calls: N files», «KB watcher initialized user/bugs: N files». Задачи НЕ создаются при первом запуске

**TC-30. Новый файл в user/bugs → задача-баг на канбане**
1. Добавь новый .md файл в https://github.com/Deci1337/mai-knowledge-base/tree/main/user/bugs
2. Подожди 5 минут (интервал KB watcher)
3. Проверь логи: должен быть «New batch: N tasks from user/bugs/...»
- Ожидание: TOP-1 получает уведомление с кнопкой [Взять на утверждение], задачи помечены как баг

**TC-31. Новый файл в user/calls → задачи из созвона**
1. Добавь новый .md файл в https://github.com/Deci1337/mai-knowledge-base/tree/main/user/calls
2. Подожди 5 минут
- Ожидание: GPT парсит транскрипт, находит задачи, TOP-1 получает уведомление

**TC-32. Кнопка [Взять на утверждение] — блокировка**
1. TOP-1 пользователь А нажимает [Взять на утверждение]
2. TOP-1 пользователь Б нажимает ту же кнопку
- Ожидание: Б получает «Задачи уже взял другой сотрудник»

**TC-33. Утверждение задач по одной**
1. Нажми [Взять на утверждение]
2. Бот показывает задачу 1/N с кнопками [Редактировать] [Утвердить]
3. Нажми [Утвердить] → показывается задача 2/N
4. Повтори до конца
- Ожидание: после утверждения всех — «Утверждено и создано N задач на канбане»

**TC-34. Редактирование задачи при утверждении**
1. На задаче нажми [Редактировать]
2. Напиши текстом: «Поменяй исполнителя на Руслана и поставь в Dev»
3. Бот показывает обновлённую карточку
4. Нажми [Утвердить]
- Ожидание: задача создана с изменёнными полями

**TC-35. Голосовое редактирование при утверждении**
1. На задаче нажми [Редактировать]
2. Отправь голосовое: «Приоритет P0, дедлайн завтра»
3. Бот распознаёт и обновляет карточку
- Ожидание: карточка обновлена, можно утвердить
