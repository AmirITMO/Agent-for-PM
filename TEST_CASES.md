## 1. Регистрация и вход (TC-1 — TC-3)

**TC-1. Регистрация нового пользователя**
1. /start → «Зарегистрироваться» → ввести имя → выбрать должность
2. Должности сгруппированы: 1 уровень / 2 уровень / 3 уровень
- Ожидание: меню с 5 кнопками (Мои задачи, Просрочки, Задать задачу, Спросить по задачам, Инструкции)

**TC-2. Вход в веб по ссылке из бота**
1. Нажми «Мои задачи» → нажми кликабельную ссылку «Доска»
2. Открывается в браузере
- Ожидание: автоматический вход без пароля, канбан-доска

**TC-3. Кнопка «Инструкции»**
1. Нажми «Инструкции»
- Ожидание: описание всех функций: кнопки меню, создание задач, управление, групповые чаты, автоматика из БЗ, напоминания, веб-трекер

## 2. Создание задач через бота (TC-4 — TC-11)

**TC-4. Простая задача — агент ВСЕГДА уточняет доску и этап**
1. «Задать задачу» → «Проверить отчёт»
2. Агент спрашивает: «На какую доску?» (кнопки: MarketAI Dev / MarketAI Marketing)
3. Выбери доску → «На какой этап?» (кнопки: Бэклог / К выполнению / В работе / Планирование)
4. Выбери этап → «Какой приоритет?» (кнопки P0-P3, т.к. приоритет не указан)
5. «Кому назначить?» (кнопки с сотрудниками)
6. Карточка → «Прикрепить файлы?» → «Нет, создать»
- Ожидание: задача на канбане, ссылка «Новая задача» кликабельная

**TC-5. Задача с именем и приоритетом — авто-заполнение**
1. «Задать задачу» → «Амиру: сделать лендинг P0»
2. Агент спрашивает доску (кнопки) → этап (кнопки)
3. Приоритет и исполнитель НЕ спрашиваются (P0 + «Амир Хайруллин» авто-заполнены)
4. Карточка: Исполнитель: Амир Хайруллин, Приоритет: P0
- Ожидание: 2 вопроса кнопками вместо 4, исполнитель — полное имя из БД

**TC-6. Голосовая задача**
1. «Задать задачу» → отправь голосовое сообщение
2. Бот: «Распознаю...» → «Распознано: [текст]» → уточняет доску/этап
- Ожидание: задача создана из голоса

**TC-7. Задача с файлами**
1. Создай задачу → на «Прикрепить файлы?» нажми «Да»
2. Отправь фото/документ → «Готово»
- Ожидание: файл в комментариях на странице задачи в вебе

**TC-8. Редактирование карточки текстом**
1. После показа карточки напиши «исполнитель Руслан» или «приоритет P0»
2. Агент обновляет карточку через GPT
- Ожидание: карточка обновлена, исполнитель = полное имя из БД (Хафизов Руслан Рустемович)

**TC-9. «Задай задачу амиру» — это задача, не напоминание**
1. «Задать задачу» → «задай задачу амиру написать мише»
- Ожидание: create_task (не set_reminder), уточняет доску/этап

**TC-10. Уведомление исполнителю при назначении**
1. Создай задачу и назначь на зарегистрированного пользователя (не себя)
- Ожидание: исполнитель получает в личку «Тебе назначена задача от [Имя]» с кликабельной ссылкой

**TC-11. Inline-кнопки: каждый шаг = одно сообщение**
1. При выборе доски → кнопки заменяют предыдущее сообщение
2. При выборе этапа → тоже
- Ожидание: в чате максимум 1 сообщение с кнопками, не спам новыми

## 8. База знаний — KB Watcher (TC-35 — TC-40)

**TC-35. KB Watcher инициализируется без спама**
1. Перезапусти контейнер: `docker-compose restart agent3-app`
2. Проверь логи: `docker-compose logs --tail=10 agent3-app`
- Ожидание: «KB watcher initialized user/calls: N files», задачи НЕ создаются при инициализации

**TC-36. Новый файл в user/bugs → задачи-баги**
1. Добавь .md файл в https://github.com/Deci1337/mai-knowledge-base/tree/main/user/bugs
2. Подожди 5 минут
- Ожидание: TOP-1 (CEO и др.) получают уведомление: список задач + кнопка «Взять на утверждение», все задачи помечены is_bug=true

**TC-37. Новый файл в user/calls → задачи из созвона**
1. Добавь .md файл в https://github.com/Deci1337/mai-knowledge-base/tree/main/user/calls
2. Подожди 5 минут
- Ожидание: GPT парсит транскрипт, извлекает задачи, TOP-1 получает уведомление

**TC-38. Утверждение задач — полный цикл**
1. Нажми «Взять на утверждение» → задача 1/N с кнопками: Редактировать / Утвердить / Удалить
2. «Утвердить» → задача 2/N → ... → все утверждены
- Ожидание: «Утверждено и создано N задач на канбане», исполнители получают уведомления

**TC-39. Редактирование при утверждении**
1. На карточке нажми «Редактировать»
2. Бот спрашивает доску/этап/приоритет/исполнителя кнопками
3. Утверди
- Ожидание: задача создана с выбранными полями

**TC-40. Удаление задачи при утверждении**
1. На карточке нажми «Удалить задачу»
- Ожидание: задача убрана из батча, показывается следующая

1. Через «Спросить» переназначь задачу с одного сотрудника на другого
- Ожидание: новый исполнитель — «Тебе назначена задача», старый — «Задача переназначена»
