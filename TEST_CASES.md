## Регистрация и вход (1-3)

**TC-1. Регистрация нового пользователя**
1. /start → «Зарегистрироваться» → ввести имя → выбрать должность
2. Должности сгруппированы: 1 уровень / 2 уровень / 3 уровень
- Ожидание: меню с 5 кнопками (Мои задачи, Просрочки, Задать задачу, Спросить по задачам, Инструкции)

**TC-2. Вход в веб по ссылке из бота**
1. Нажми «Мои задачи» → скопируй ссылку «Открыть трекер»
2. Открой в браузере
- Ожидание: автоматический вход, канбан-доска

**TC-3. Кнопка «Инструкции»**
1. Нажми «Инструкции»
- Ожидание: описание всех функций включая групповые чаты, автоматику из БЗ, напоминания

## Создание задач через бота (4-9)

**TC-4. Текстовая задача — агент уточняет доску и этап**
1. «Задать задачу» → «Сделать лендинг для Амира P0»
2. Агент спрашивает: «На какую доску?» (с кнопками)
3. Выбери доску → агент спрашивает этап (с кнопками)
4. Выбери этап → карточка + файлы? → «Нет, создать»
- Ожидание: задача на канбане, ссылка кликабельная «Открыть задачу»

**TC-5. Голосовая задача**
1. «Задать задачу» → отправь голосовое
2. Бот: «Распознаю...» → «Распознано: ...» → уточняет доску/этап
- Ожидание: задача создана из голоса

**TC-6. Задача с файлами**
1. Создай задачу → на «Прикрепить файлы?» нажми «Да»
2. Отправь фото → «Готово»
- Ожидание: фото в комментариях на странице задачи

**TC-7. Агент НЕ создаёт без доски и этапа**
1. «Задать задачу» → «Проверить отчёт»
- Ожидание: агент спрашивает доску, НЕ создаёт автоматически

**TC-8. «Задай задачу амиру» — это задача, не напоминание**
1. «Задать задачу» → «задай задачу амиру написать мише»
- Ожидание: create_task (не set_reminder), уточняет доску

**TC-9. Уведомление исполнителю при назначении**
1. Создай задачу и назначь на зарегистрированного пользователя
- Ожидание: исполнитель получает в личку «Тебе назначена задача от ... — Открыть задачу»
